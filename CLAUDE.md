# Aurora Arcology — Claude Code instructions

> Project-specific instructions. Repo-level overrides global guidance.

## Project

**Aurora** — investigation-board framework for narrative universes. First instance:
EVE Online lore. Spec: `aurora-spec-v0.4.md`. Phase 0 execution: `aurora-phase0-execution-v1.1.md`.

The data model is the load-bearing decision (spec §5):
- Three primary entity types: nodes, sources, connections
- Connections are **first-class objects** with claim text, confidence, curator, supporting/contested-by sources
- Vocabulary tables (`node_types`, `relation_types`) are editable at runtime

## Stack

- Next.js 15 + TypeScript + Tailwind
- better-sqlite3 + Drizzle ORM (synchronous, single-process, local-first)
- Zod for runtime validation at the curator UI ↔ DB boundary
- D3 force-directed graph (Phase 1 — current Phase 0 view is a list placeholder)

## Conventions

### Provenance is mandatory

Every claim originates from a source citation. When adding a node, populate `sources`.
When adding a connection, populate either `supporting_sources` or `contested-by`.
When in doubt, downgrade `confidence` and add the contested-by note rather than dropping
the claim.

### Confidence calibration

- `≥ 0.9` — direct CCP source quote / paraphrase
- `0.7–0.9` — creator's structural read backed by multiple CCP sources
- `0.5–0.7` — creator interpretation; defensible on stream
- `< 0.5` — speculation. Always carries a `contested-by` note. Filter out by default.

### L3 user-paste licensing

Aurora does not redistribute creator transcripts. Raw transcripts and captions stay
under `data/raw/` (gitignored). Only curator-authored seed JSON (paraphrased claim
text + source citation metadata) is committed.

### Append-only audit log

Every node, source, and connection insertion writes to `audit_log`. Do not bypass
this — Tier 4 temporal-layer queries depend on it.

## Don't

- Don't hardcode node types or relation types as TypeScript enums. They live in the
  vocabulary tables (`node_types`, `relation_types`). Reference by string ID; let
  the curator extend.
- Don't reproduce verbatim transcript chunks in seed files. Curator-paraphrased claim
  text only. Brief verbatim quotes (a sentence or two) are fine for confidence calibration.
- Don't skip the audit log when writing.
- Don't mutate connections in place — supersede them with a new connection and let the
  audit log reflect the change.
